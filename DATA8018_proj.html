The quest for generating high-fidelity 3D assets—combining both geometry and texture—represents one of the most exciting frontiers in computer vision and graphics. This survey explores the evolution from 2D prior-based generation to native 3D generative models, showcasing breakthrough approaches that enable unprecedented quality and scalability.

Part 1: The Shift to "Native" 3D

🖼️ 2D Prior-Based Methods

Historical approaches like DreamFusion and Score Distillation:

  • Leverage pre-trained 2D image models
  • Use score distillation to guide 3D optimization
  • Suffer from the "Janus Problem" (multi-faced objects)
  • Computationally expensive and slow
  • Key limitation: Indirect 3D representation

🎯 Native 3D Generation

The modern paradigm—generating 3D directly in 3D space:

  • Direct Generation: Predicting points/voxels directly
  • Latent Generation: Learning compressed representations
  • Dramatically faster inference (seconds vs. minutes)
  • Superior quality and consistency
  • Key insight: 3D works best in 3D

⚡ The Core Challenge

3D data is fundamentally different from 2D images: it is sparse (surfaces occupy only a small fraction of space), has varying topology, and requires efficient representations to be tractable for generative models. Native 3D generation tackles this by learning latent spaces tailored specifically for 3D data, enabling both high compression and high-quality reconstruction.

Part 2: Deep Dive—Latent 3D Generation

📐 3DShape2VecSet: The Foundation—Vectors for Generative Models

3DShape2VecSet addresses a fundamental question: How should we represent 3D shapes for generative diffusion models? Rather than using voxel grids or directly optimizing point clouds, this work introduces a learned vector set representation optimized for transformer architectures.

Core Architecture:

  • Cross-Attention Encoder: Input surface/point cloud → cross-attention with ~1K-4K learnable query vectors
  • Learnable Queries: Transformer-based query tokens learn to extract shape information
  • Neural Field Decoder: Query vectors → coordinate-based neural field for continuous surface
  • Diffusion-Ready: Vector set is compact and transformer-native for diffusion training

Quantitative Results (ShapeNet):

IoU 0.965
Chamfer Distance 0.026
F-Score 0.999

Significance: Near-perfect reconstruction (F-Score: 0.999) with compact representation

🔑 Key Innovations:

  • First to use vector sets for 3D generation
  • Cross-attention instead of spatial convolutions
  • Extremely compact (1K-4K vectors vs millions of voxels)
  • Supports diverse conditioning (text, image, unconditioned)

📊 Achievements:

  • Beats OccNet, ConvOccNet, IF-Net on IoU/Chamfer/F-Score
  • Point completion with high accuracy
  • Text-to-3D generation quality
  • Fast inference (transformer-native processing)

🎨 Michelangelo: The Multi-Modal Revolution

Just as Stable Diffusion revolutionized 2D generation with a CLIP-aligned latent space, Michelangelo applies the same principle to 3D: align 3D shapes with vision-language models. This enables semantic understanding and powerful conditional generation.

Core Architecture:

  • SITA-VAE: Shape-Image-Text Alignment VAE
  • Three Encoders: Shape encoder (PointNet), CLIP image encoder, CLIP text encoder
  • Shared Latent: All three modalities map to aligned 512-D tokens
  • Neural Field Decoder: Reconstructs shape from latent tokens
  • Diffusion in Latent Space: ASLDM generates new shapes in aligned space

Multi-Modal Performance:

Text-to-3D (ST-S) 16.647
Image-to-3D (SI-S) Best
Coverage ×3 improvement

Key: Alignment with CLIP dramatically improves semantic control and diversity

🔑 Key Innovations:

  • First to align 3D with CLIP embeddings
  • Multi-modal contrastive learning for 3D
  • Domain gap reduction via vision-language alignment
  • Unified generation for text/image conditioning

📊 Achievements:

  • SOTA text-to-3D generation quality
  • SOTA on image-to-3D task
  • 3× coverage improvement (diversity)
  • Semantic alignment with language

Michelangelo demonstrates multi-modal generation from text, images, and shape descriptions

🏗️ TRELLIS: Structured Latents for Versatility

Where VecSet prioritizes compression, TRELLIS (Structured LATents) prioritizes spatial structure and versatility. The breakthrough insight: if we know where surface points should be in 3D space, we can decode to multiple formats from the same latent representation.

Core Architecture:

  • Sparse 3D Grid: 16³ or 64³ voxel grid with localized latents only at surface voxels
  • SLAT (Structured Latents): Each voxel stores 64-D local feature vectors
  • Fusion: Combines multiview image features from vision foundation model (e.g., DINO)
  • Multi-Format Decoders: Same latents → 3D Gaussians, NeRFs, or mesh surfaces
  • Rectified Flow Transformer: 2B parameter model generates SLAT tokens

Generation Speed & Quality:

Speed (10s generation) ~10 seconds
Model Scale 2B parameters
Train Data 500K objects

Versatility: Gaussian, NeRF, or mesh output from same model

🔑 Key Innovations:

  • Sparse voxel grid with localized latents
  • Multi-format output decoding
  • Large-scale training (2B params, 500K data)
  • Editing via latent manipulation
  • Vision foundation model fusion (DINO)

📊 Achievements:

  • SOTA generation quality on diverse objects
  • Coherent Gaussian, NeRF, and mesh outputs
  • Text and image-conditioned generation
  • Local 3D editing capabilities
  • Production-ready pipeline

TRELLIS demonstrates versatile outputs and user interaction capabilities

🚀 LATTICE: The Scaling Breakthrough

LATTICE unifies VecSet's compression with TRELLIS's spatial structure through an elegant innovation: voxel-anchored query tokens. Unlike TRELLIS (only surface voxels), LATTICE uses a dense grid with localizable positions known at test time—enabling dynamic resolution scaling.

Core Architecture:

  • VoxSet Representation: Tokens anchored at 64³ (or higher) voxel grid positions
  • Localizable Queries: Position information known at inference (unlike learned queries)
  • RoPE Positional Embeddings: Enables extrapolation to higher resolutions
  • Test-Time Scaling: 6K tokens at train time → 12K+ at inference
  • Large Scale Model: 4.5B parameters trained on massive datasets

Scaling Performance:

Model Size 4.5B params
Train Tokens ~6K
Inference Scaling 6K → 12K+ tokens
Scaling Gains Consistent improvement

Breakthrough: Test-time resolution scaling without retraining

🔑 Key Innovations:

  • Voxel-anchored query tokens (localizable)
  • RoPE positional embeddings for scaling
  • Test-time resolution extrapolation
  • Hybrid compression + structure approach
  • Massive scale training (4.5B params)

📊 Achievements:

  • SOTA high-fidelity generation quality
  • Dynamic resolution scaling capability
  • Consistent scaling benefits up to 4.5B
  • Pareto frontier: compression + structure
  • Best of both VecSet and TRELLIS

Part 3: Bridging Generation & Reconstruction—CUPID & SAM 3D

🎭 The Fundamental Tension

✨ Pure Generative Models

  • 💡 Imagine diverse possibilities from scratch
  • 🎨 High creativity and unbiased priors
  • ⚠️ Can produce geometrically inconsistent shapes
  • ❌ May neglect details visible in input image
  • ❌ Struggle with texel-level consistency

🎯 Pure Reconstruction Models

  • 🔍 Faithfully preserve observed visual details
  • 📐 Geometrically accurate and consistent
  • ⚠️ Cannot hallucinate occluded regions
  • ❌ Limited by input image resolution/quality
  • ❌ Cannot improve on poor input images

The solution: jointly model both aspects through explicit geometric reasoning (pose estimation) or multi-stage refinement.

🎀 CUPID: Pose-Aware Generative Reconstruction

CUPID bridges generation and reconstruction by explicitly modeling camera pose and object geometry jointly. The key insight: if we know where the camera is relative to the object, we can directly inject image features into the correct 3D locations.

Two-Stage Pipeline:

Stage 1: Pose Estimation & Rough Geometry

  • Generate occupancy grid (rough shape)
  • Generate UV cube (3D-to-2D surface correspondences)
  • Solve camera pose via PnP (Perspective-n-Point)
  • Output: Estimated camera matrix and approximate geometry

Stage 2: Pixel-Aligned Refinement

  • Generate high-fidelity appearance & geometry features
  • CRITICAL: Inject pixel-aligned features from input
  • Prevents "color drift" (texture-ground-to-3D drift)
  • Preserves fine details visible in input image

Technical Achievements:

Color Consistency Preserved
Geometry Quality SOTA
Input Faithfulness High

Key Innovation: Explicit pose modeling enables pixel-to-3D alignment, solving the core weakness of pure generative models.

🔑 Key Innovations:

  • Joint pose + geometry generation
  • UV cube for 3D-2D correspondence
  • PnP-based pose estimation
  • Pixel-aligned feature injection
  • Color drift elimination

📊 Achievements:

  • SOTA generative reconstruction quality
  • Faithful texture preservation
  • High geometric accuracy
  • Superior on benchmark datasets
  • Hybrid approach benefits

🚀 SAM 3D: Scale and Real-World Grounding

Inspired by the success of SAM (Segment Anything) in 2D vision, SAM 3D applies massive-scale learning to 3D generation. The philosophy: if we train on diverse real-world 3D data at scale, we learn robust priors for handling occlusion, clutter, and complex scenes.

Model-in-the-Loop Paradigm:

  • Stage 1: Coarse 3D layout prediction (bounding box, rotation, scale)
  • Stage 2: Per-object refinement (shape and texture)
  • Stage 3: Fine-grained surface details and appearances
  • Learning Signal: Human feedback + synthetic data augmentation
  • Scale: Trained on massive real-world 3D datasets (100K+)

Real-World Performance:

Occlusion Handling Excellent
Real Image Quality SOTA
Multi-Object Scenes Supported
Cross-Dataset Gen. Strong

Strength: Real-world grounding through massive diverse training data

🔑 Key Innovations:

  • Multi-stage coarse-to-fine pipeline
  • Human-in-the-loop data collection
  • Massive-scale training (100K+ shapes)
  • Real-world occlusion handling
  • Multi-modal (shape + texture)

📊 Achievements:

  • SOTA real-world image-to-3D
  • Handles cluttered scenes well
  • User preference rankings (ELO)
  • Cross-dataset generalization
  • Robust to diverse input quality

Part 4: Takeaways -- From Compression to Structure to Scaling

The Evolution of 3D Representations

Phase 1: Compression (VecSet)

Query tokens + cross-attention maximize compression. Enables fast inference and transformer processing.

  • Size: 1K-4K tokens
  • Strength: Compact, efficient
  • Weakness: No spatial structure

Phase 2: Structure (SLAT)

Sparse 3D grids anchor latents to spatial positions. Enables multi-format decoding and editing.

  • Size: ~20K tokens (64³ sparse)
  • Strength: Spatial information
  • Weakness: Fixed resolution

Phase 3: Scaling (VoxSet)

Voxel-anchored queries with RoPE embeddings enable test-time scaling. Combines compression + structure.

  • Size: 6K-12K+ tokens (scalable)
  • Strength: Pareto frontier
  • Weakness: Training complexity

1. Foundation: Vector Sets Work for 3D

3DShape2VecSet proved that unstructured vector sets—rather than voxel grids—are an effective 3D representation. Cross-attention queries compress 3D surfaces into compact tokens perfect for transformers. Quantitative proof: F-Score of 0.999 on ShapeNet reconstruction (vs OccNet: 0.858, IF-Net: 0.967).

2. Alignment: Semantic Understanding Matters

Michelangelo introduced the crucial insight that aligning 3D with vision-language models (CLIP) dramatically improves semantic control. 3× improvement in coverage and best-in-class text-to-3D generation. The SITA-VAE bridges the modality gap through contrastive learning.

3. Structure: Sparse Grids Enable Versatility

TRELLIS proved that adding spatial structure via sparse 3D grids dramatically improves downstream task performance. Same latents can decode to 3D Gaussians, NeRFs, or meshes—enabling rendering, editing, and diverse applications. Scale reaches 2B parameters on 500K objects.

4. Scaling: Positional Information Enables Extrapolation

LATTICE achieves the Pareto frontier by combining compression (VecSet) with structure (SLAT) through voxel-anchored queries. Crucially, RoPE positional embeddings enable test-time resolution scaling (6K→12K tokens) without retraining. Models up to 4.5B parameters show consistent scaling benefits.

5. Alignment: Explicit Geometry Bridges Generation & Reconstruction

CUPID solves a fundamental problem: how to combine generative priors with reconstruction faithfulness. By explicitly modeling camera pose and enabling pixel-aligned feature injection, CUPID achieves state-of-the-art reconstruction quality while preserving generative diversity and texture consistency.

6. Data at Scale: Real-World Grounding Drives Performance

SAM 3D demonstrates that massive-scale training on real-world 3D data (100K+) with human-in-the-loop refinement produces robust models that generalize well. Multi-stage coarse-to-fine generation and per-object refinement handle occlusion and clutter naturally, achieving SOTA on real image-to-3D tasks.

Part 5: Future Research Directions

🏙️ Scene-Level Generation

Current methods excel at single objects. The frontier is entire scenes—rooms, outdoor environments, and complex compositions.

  • ✓ Multi-object relationships
  • ✓ Spatial coherence & constraints
  • ✓ Physical plausibility
  • ✓ Semantic consistency

🦾 Articulated Objects

Beyond static models: generating objects with joints, limbs, and moving parts.

  • ✓ Rigged character generation
  • ✓ Semantic part understanding
  • ✓ Motion-aware representations
  • ✓ Animation-ready assets

📐 Mesh Topology

From "soup of triangles" to functionally correct manifold meshes for professional workflows.

  • ✓ Watertight, manifold meshes
  • ✓ Quad-based topology
  • ✓ Production-ready output
  • ✓ Topological consistency
